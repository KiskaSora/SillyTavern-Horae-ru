Основные настройки
После отключения иконка скрывается без пустого места. Повторно включить — через «Horae — Память Времени» в панели расширений (иконка конструктора).
Ширина панели в % от области чата (50–100). Для широкоэкранных тем можно уменьшить. Смещение регулирует горизонтальное положение.
Внешний вид
Выберите встроенную или импортированную тему. После импорта .json-файла она появится в списке.
Пользовательский CSS
Введите CSS для переопределения стилей плагина. Изменения вступают в силу после смены фокуса.
Расширенные настройки
Количество «обычных» событий, отправляемых ИИ:
0 = не отправлять обычные (важные/ключевые отправляются)
15 = отправлять последние 15 обычных событий (по умолчанию)
Чем больше число — тем больше расход токенов
Позиция вставки данных Horae в промпт ИИ:
0 = самый конец (вплотную к последнему сообщению)
1 = предпоследняя позиция (рекомендуется)
Чем больше число — тем выше позиция
Пользовательские таблицы
Создавайте таблицы в стиле Excel — ИИ заполняет их по вашим правилам.
Удерживайте заголовок для добавления/удаления строк и столбцов.
Локальные — только для текущего диалога; Глобальные — общие для всех.
🔒 Блокировка защищает содержимое от изменений ИИ.
Отправляемые ИИ данные
Выберите, какие данные отправляются с промптом. Отключение экономит токены.
Инъекция токенов:Вычисляется...
Конспекты событий, относительное время. После отключения ИИ не знает, что было «несколько дней назад».
Присутствующие персонажи, описания нарядов, симпатия, информация о персонажах. После отключения ИИ может описывать неверные наряды.
Текущие предметы и их местонахождение. После отключения ИИ может «достать из воздуха» несуществующий предмет.
Фиксирует постоянные характеристики мест. После включения ИИ запоминает место при первом визите.
Отслеживает изменения отношений (друзья, возлюбленные, враги). ИИ выводит только при изменении. После отключения тег rel: и промпт не инъектируются.
Отслеживает изменения эмоций персонажей. ИИ выводит только при явных изменениях.
После включения ИИ выводит атрибуты (HP/MP/SP) и навыки в тегах <horaerpg>. При отключении — нулевой расход токенов.

Авто-конспект и скрытие
При превышении лимита старые события сжимаются ИИ в конспект — экономит токены.
Конспект можно переключить обратно на исходные события в таймлайне.
События последних N сообщений не сжимаются — сохраняется детализация недавнего сюжета.
Авто-сжатие запускается при превышении этого значения.
Максимум сообщений за один проход. Старые файлы обрабатываются постепенно.
Лимит токенов текста за один проход; превышение усекается.
Рекомендуется 30–50% от окна контекста модели:
Claude ≈ 80K~200K · GPT-4o ≈ 80K~128K · Gemini ≈ 100K~1000K · DeepSeek ≈ 64K
Укажите OpenAI-совместимый эндпоинт — конспект будет генерироваться через него, не занимая основное соединение.
Базовый адрес OpenAI-совместимого эндпоинта, напр. https://api.openai.com/v1
Подойдёт бюджетная модель: gpt-4o-mini, deepseek-chat и т. д.
Авто-конспект будет генерироваться через независимый HTTP-запрос, не затрагивая основной API таверны.
Поддерживаются все OpenAI-совместимые эндпоинты (OpenAI, DeepSeek, OpenRouter и др.).
Тест подключения только запрашивает список моделей — генерации не расходуются.
Векторная память (Бета)
Векторный поиск по данным Horae с автоматическим извлечением релевантных фрагментов.
Локальный режим: скачивает ~25 МБ модели в кэш браузера, занимает 50–80 МБ памяти. ⚠ Не рекомендуется на телефонах менее 2 ГБ. Убедитесь, что задан --max-old-space-size.
API-режим: использует удалённый Embedding API без локальных ресурсов.
q4 подходит большинству устройств; q8 точнее, но вдвое больше.
Только модели Embedding (эмбеддинг/векторные)!
Не вводите LLM-чат-модели (gemini, gpt-4 и т. д.) — будет ошибка.
Рекомендуются: text-embedding-v3 (Qwen), text-embedding-3-small (OpenAI)
OpenAI-совместимый формат — путь /embeddings добавляется автоматически.
Пример: https://dashscope.aliyuncs.com/compatible-mode/v1
Qwen: text-embedding-v3 / text-embedding-v2
OpenAI: text-embedding-3-small / text-embedding-3-large
Для высокомерных API-моделей (Qwen3-Embedding, text-embedding-3-large и т. д.).
После включения отключаются расширение ключевых слов, сопоставление фраз, штраф за частотность и другие эвристики.
Полностью полагается на семантику модели для уменьшения шума.
Для малых или низкомерных моделей не включайте — качество ухудшится.
После векторного поиска Rerank-модель делает точную сортировку кандидатов.
Релевантное поднимается выше, нерелевантное — ниже; ИИ получает более точные воспоминания.
Требует отдельной настройки API Rerank-модели:
Рекомендуемые модели: Qwen3-Reranker, bge-reranker-v2-m3, bce-reranker
Qwen: gte-rerank / gte-rerank-v2
Jina: jina-reranker-v2-base-multilingual
По умолчанию Rerank использует конспект. После включения — полный текст сообщений.
Улавливает детали, пропущенные в конспекте, но требует большого контекстного окна модели.
Только для длинноконтекстных Rerank-моделей (Qwen3-Reranker поддерживает 32K).
Для короткоконтекстных (bge-reranker-v2-m3 ≤1024, bce-reranker ≤512)
не включайте — полный текст усечётся и качество ухудшится.
При каждой отправке извлекается K наиболее релевантных воспоминаний и инъектируется в контекст.
Воспоминания с низким сходством не извлекаются. Повышение уменьшает шум.
Топ-N результатов выше порога отправляют исходный текст, а не конспект. 0 — отключить.
Только результаты выше этого порога отправляют полный текст, избегая лишних токенов.
