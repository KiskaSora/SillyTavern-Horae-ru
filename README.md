# Horae - Память Времени v1.8.4 | Плагин расширения памяти для SillyTavern

![Image](https://github.com/SenriYuki/SillyTavern-Horae/blob/main/HoraeLogo.jpg)

> *Horae (Хорай) — богини времён года и порядка в греческой мифологии*

С проблемой, с которой сталкиваются все игроки в длинные RP, вы наверняка знакомы — память ИИ как у золотой рыбки: вчерашние события называет сегодняшним утром, а всё, что было несколько дней назад, вечно «вчера»; в прошлой сцене — школьная форма, в следующей — вдруг повседневная одежда; отношения с NPC переворачиваются с ног на голову; подаренный предмет бесследно исчезает, а выброшенный — снова оказывается в руках.

**Horae использует структурированные временны́е якоря, чтобы дать вашему ИИ надёжный дневник памяти.**

---

## Ключевые возможности

**Режим RPG — полоски здоровья, навыки и статусы с первого взгляда (новое)**
RPG-система, созданная специально для карточек в жанре западного фэнтези / даосского культивирования / боевых историй. После включения ИИ выводит атрибуты персонажа (HP/MP/SP и пользовательские характеристики), отрицательные статусы и изменения навыков в тегах `<horaerpg>`. Над нижней панелью сообщений автоматически отображается упрощённый HUD со статусными полосками, а на верхней панели появляется новая вкладка RPG с подробными атрибутами и списком навыков. Названия и цвета полосок можно настраивать, навыки связаны с NPC-персонажами. При отключении режима — никаких промптов и расхода токенов. Поддерживается отдельная настройка цвета фона и прозрачности RPG-панели в инструменте кастомизации.

**Векторный движок памяти — возвращаем потерянные детали (новое)**
Умный внешний мозг, специально разработанный для использования совместно с «Авто-конспектом и скрытием»! Пропали детали после сжатия многих сотен тысяч слов длинного сюжета? Теперь, когда диалог касается исторических событий, плагин автоматически извлекает нужные фрагменты из скрытых старых временны́х линий. Всё вычисляется локально через Web Worker — нулевой расход API, плавная работа без подтормаживания интерфейса (при первом запуске необходимо скачать кэш модели ~30–60 МБ; доступны две модели — bge с оптимизацией для китайского и многоязычная e5 ⚠ не рекомендуется на телефонах с памятью до 2 ГБ или в облачных тавернах).

**Навигация для новичков — даже первый раз не заблудитесь (новое)**
При первом использовании Horae автоматически запускается интерактивное обучение, которое поэтапно знакомит со всеми функциональными областями и настройками. От базовых концепций до продвинутых функций (глубина контекста, позиция инъекции, пользовательские промпты и т. д.) — всё за один раз. Опытные пользователи могут в любой момент нажать «Перезапустить обучение» в настройках.

**Инструмент кастомизации — настраивайте без знания CSS (новое)**
Визуальная панель кастомизации для пользователей без опыта программирования. Быстрая настройка цвета с помощью ползунков оттенка, насыщенности и яркости, переключение между дневным и ночным режимами, импорт URL-адресов изображений для оформления шапки, фона ящика и нижней панели. Все изменения отображаются в реальном времени — сохраните, когда понравится, и экспортируйте для обмена. Разумеется, для продвинутых пользователей сохранены импорт/экспорт JSON-тем и написание CSS вручную. (⚠️ Для создания кастомизации плагина Horae моё разрешение не нужно — делайте как хотите.)

**Память сцены — таверна больше не меняется за ночь (новое)**
ИИ описывает одно и то же место, а в прошлый раз там был камин, а в этот — нет? Память сцены записывает постоянные физические характеристики места, автоматически подставляя их при следующем посещении. Умное сопоставление названий («Таверна·Зал» автоматически откатывается до «Таверна»), отправляется только совпавшее описание, при отсутствии совпадения — нулевая инъекция.

**Эмоции и сеть отношений — ИИ понимает межличностный контекст (новое)**
Отслеживание эмоций позволяет ИИ помнить текущее психологическое состояние персонажа — больше не будет «плакал секунду назад, смеётся сейчас». Сеть отношений фиксирует тип и изменения связей между персонажами — враг больше никогда не станет другом. Оба компонента управляются изменениями: при отсутствии изменений — нулевой вывод.

**Авто-конспект — экономия токенов в длинных историях**
Сотни сообщений и всё ещё отправляете полным объёмом? Авто-конспект автоматически сжимает старые сообщения в краткий конспект в фоне, скрывая исходные сообщения командой `/hide` для экономии токенов. Конспект и исходная временна́я линия переключаются в одно касание — что видите, то и отправляется. Можно подключить отдельный бюджетный **независимый API** для настоящей параллельной работы — конспект генерируется без занятия основного соединения.

**Векторная память — пусть ИИ вспомнит скрытые детали (новое)**
Умная система воспоминаний для использования совместно с авто-конспектом. После сжатия детали старых сообщений не теряются — когда диалог касается исторических событий, плагин автоматически извлекает релевантные фрагменты из скрытой временно́й линии и инъектирует в контекст. Всё вычисляется **локально**, без дополнительного расхода API. Поддерживаются модели с оптимизацией для китайского и многоязычная.

**ИИ-конспект**
Ранее не использовали Horae? Одним нажатием сканируйте все исторические сообщения — ИИ автоматически извлечёт события сюжета и изменения предметов и сформирует полную временну́ю линию. Поддерживается настройка максимального числа токенов на пакет для разных моделей; при неудовлетворительном результате — отмена в одно касание.

**Прощай, «вечное вчера»**
Плагин автоматически вычисляет относительное время и инъектирует живые формулировки: «вчера», «в прошлую среду», «15-го прошлого месяца». ИИ наконец-то начнёт различать позавчера и прошлую неделю. Поддерживаются современный, исторический, фэнтезийный и другие календарные системы.

**Одежда больше не путается**
Текущий наряд каждого персонажа зафиксирован и записан, отправляются только наряды присутствующих персонажей. Персонаж больше не наденет вдруг вчерашнюю одежду, а ИИ не будет тратить токены на наряды отсутствующих.

**NPC не будут размываться с каждым разом**
Внешность, характер, отношения и другие поля отслеживаются независимо. Возраст автоматически увеличивается со временем сюжета. Строго прописанный промпт для отношений: больше не будет «секунду назад Char должен User деньги, а в следующей реплике ИИ пишет, что User должен Char».

**Инвентарь наконец-то надёжен**
Система предметов с уникальными идентификаторами, три уровня важности: обычный/важный/ключевой. Умный разбор изменений количества (5 кг → 4 кг), автоматическое обнаружение статусов вроде «потреблён» и удаление.

**Задачи не забудутся**
ИИ может автоматически записывать договорённости и незакрытые сюжетные линии, удаляя их по выполнении. Никаких опасений, что ИИ забудет свидание, назначенное две недели назад.

**Таблицы уровня «детское питание» по кастомизации**
Свободные таблицы в стиле Excel — добавляйте строки и столбцы сколько угодно, прикрепите промпт для автозаполнения ИИ. Расписания, отношения персонажей, списки задач — всё, что вы придумаете. При удалении сообщения заполненные ИИ данные откатываются, введённые вами — всегда сохраняются.

**Управляемый изменениями, экономит токены**
ИИ выводит только информацию, изменившуюся в этом ходу, не повторяя каждый раз весь статус. Каждый тип данных имеет чёткие правила срабатывания, сокращая бесполезный вывод. **Не требует дополнительных генераций** — плагин автоматически разбирает узлы памяти при обычных ответах ИИ, нулевые дополнительные расходы.

**Не боится усечения и перезаписи**
Сюжет был усечён посередине или вы вручную переписали часть? Встроенный **анализ ИИ** в нижней панели позволяет умно заполнить узлы памяти для этого сообщения всего за одну генерацию.

**Простой интерфейс, дружелюбный к новичкам**
Нижняя панель понятна с первого взгляда, вкладки верхней панели чётко разделены. Сложная настройка не нужна — готово к работе после установки.

**Свободный контроль отправляемых данных**
В настройках выбирайте, какие данные отправлять ИИ, а какие нет. Не нужно отслеживание персонажей? Отключите. Не нужен инвентарь? Отключите. Комбинируйте по потребности и экономьте токены.

## Быстрая установка

1. Откройте SillyTavern → Панель расширений (иконка конструктора) → «Установить расширение»
2. Вставьте Git-ссылку на этот репозиторий и нажмите «Установить»
3. После установки обновите страницу

> Сопутствующий регекс будет **автоматически добавлен** при первой загрузке плагина — ручной импорт не требуется.

## Что нового в v1.1.0

- **ИИ-конспект**: пакетный анализ исторических сообщений, автоматическое извлечение событий и предметов, настраиваемый лимит токенов на пакет, отмена в одно касание
- **Счётчик токенов**: отображает общий объём промпта, отправляемого ИИ; расположен под настройками. К слову, при стандартной работе плагин сейчас отправляет 3 591 токен
- **Адаптация свайпа**: исправлена проблема, когда при генерации нового свайпа старые данные памяти переносились в новый. Теперь при генерации свайпа старая память текущей ветки автоматически исключается, ИИ генерирует новую ветку на основе правильного контекста
- **Совместимость с кастомизацией**: стили интерфейса плагина изолированы от внешних тем, больше нет чёрного текста на чёрном или белого на белом
- **Изменение регекса**: `<horaeevent>` тоже настроен как не отправляемый ИИ — экономит токены
- **Уточнение описаний**: некоторые описания функций исправлены для большей точности

## Что нового в v1.3.0

- **Глобальные таблицы**: в пользовательских таблицах появились два режима — «Глобальный» и «Локальный». Глобальные таблицы доступны всем карточкам персонажей, идеальны для фиксированных данных (например, таблица ID персонажей). Переключение — нажатием на иконку перед именем таблицы
- **Блокировка строк и столбцов**: правый клик / долгое нажатие на ячейку позволяет заблокировать строку, столбец или отдельную ячейку от изменений ИИ. Заблокированный контент помечается 🔒 в промпте, ИИ пропускает запись в него
- **Очистка таблицы**: в панели операций с таблицей появилась кнопка-ластик, очищающая все данные, но сохраняющая заголовки — удобно для перезаполнения ИИ
- **Система тем**: три встроенные темы — «Сакурово-розовая», «Лесная зелень», «Морская синь» — выбираются из списка. Поддерживается импорт/экспорт пользовательских JSON-файлов тем для обмена в сообществе
- **Навигация по кастомизации**: добавлен документ `THEME.md` со всеми CSS-переменными, основными именами классов контейнеров и форматом файлов кастомизации
- **Дневной/ночной режим**: переключается в настройках, влияет только на интерфейс плагина
- **Пользовательский CSS**: в настройках можно вводить дополнительный CSS-код для тонкой настройки стилей
- **Исправление контекстного меню**: устранена прозрачность контекстного меню (CSS-переменные не охватывали элементы уровня `document.body`)
- **Усиленная изоляция стилей**: все кнопки заменены на классы, эксклюзивные для плагина; иконки верхней панели больше не меняются в светлом режиме
- **Ширина панели сообщений**: в настройках можно задать ширину панели сообщений (50–100%)
- **Регекс в конце**: регекс плагина при запуске автоматически перемещается в конец списка, уменьшая конфликты с другими регексами
- **Мультивыбор в таймлайне**: долгое нажатие на событие активирует мультивыбор для пакетного удаления (время срабатывания специально увеличено, чтобы избежать случайных нажатий)

## Что нового в v1.4.0

- **Поддержка десятичных значений симпатии**: симпатия теперь полностью поддерживает десятичные числа, совместимо с системами на основе World Book, использующими дробные шаги

- **Улучшение мультивыбора в таймлайне**: теперь мультивыбор переключается кнопкой в верхней части (прежнее долгое нажатие изменено на меню вставки)
- **Страницы конспекта**: новый тип события «конспект» — без отображения временно́й метки, пустой статус подсказывает «Не удаляйте начальную временну́ю линию»

- **Улучшение понимания таблиц ИИ**: данные таблиц, передаваемые ИИ, теперь содержат координатные метки (например, `[1,2]содержимое`) и объявление размеров, существенно снижая ошибки ИИ при заполнении
- **Исправление очистки данных таблиц**: при удалении таблицы или очистке одним нажатием теперь синхронно удаляются все исторические записи ИИ (tableContributions) и базовые снимки (baseData) во всех сообщениях, полностью исключая возврат старых данных

## Что нового в v1.5.0

Обновление посвящено «детализации» и «экономии токенов в длинных историях».

### Новые функции

- **Отслеживание эмоций/психологического состояния**: отслеживание изменений эмоций присутствующих персонажей (например, «напряжён/тревожен», «радостен/в предвкушении»). Нижняя панель автоматически отображает метки текущих эмоций персонажа. **Минимальный расход токенов** — ИИ выводит тег `mood:` только при явном изменении эмоций (обычно 10–20 токенов), нулевой вывод при отсутствии изменений. Переключается в настройках
- **Сеть отношений**: запись отношений между персонажами (друзья, возлюбленные, начальник/подчинённый, враги и т. д.). Список отношений отображается в нижней части страницы NPC, поддерживается ручное редактирование. Отображение отношений в нижней панели **полностью рендерится плагином из базы данных, без расхода токенов ИИ**. Со стороны ИИ — управление изменениями: тег `rel:` выводится только при изменении отношений, нулевой вывод для уже зафиксированных. Переключается в настройках
- **Память сцены**: запись постоянных физических характеристик мест для сохранения единообразия описаний между ходами. **Умный поиск** — плагин сопоставляет записанные описания сцен с текущим названием места и отправляет только совпавшее (обычно 50–150 знаков), а не все записи мест. Нулевая инъекция при отсутствии совпадений. Поддерживается обратный поиск для составных названий («Таверна·Зал» → «Таверна»)
- **Авто-конспект и скрытие**: включение в настройках автоматически запускает полный конспект при превышении порога несжатых сообщений. После завершения исходные сообщения скрываются `/hide` для экономии токенов, в таймлайне появляется карточка конспекта. Переключение между «режимом конспекта» и «исходной временно́й линией» — одним нажатием
- **Независимый API (настоящий параллелизм)**: для авто-конспекта можно настроить отдельный OpenAI-совместимый эндпоинт (достаточно ввести адрес API / ключ / имя модели). Генерация конспекта работает через прямой HTTP-запрос, не занимая основное соединение таверны. Поддерживаются все совместимые эндпоинты: OpenAI, DeepSeek, OpenRouter и другие
- **Кнопка отмены сжатия**: добавлена кнопка отмены для сжатия событий и полного конспекта — нажатие немедленно прерывает запрос к бэкэнду (аналогично кнопке отмены в ИИ-конспекте)
- **Наследование данных таблиц при ветвлении**: при создании ветки или откате сообщений таблица наследует только данные ИИ и пользователя вплоть до этого узла. Например, при создании ветки на #10 наследуются только данные таблицы до #10 включительно

### Улучшения

- **Разделение данных глобальных таблиц по карточкам**: структура глобальных таблиц (заголовки, имена, промпты, блокировки) общая для всех карточек, но данные заполнения хранятся отдельно для каждой. Переключение карточек больше не переносит данные других карточек
- **Умное сжатие памяти**: в режиме мультивыбора в таймлайне можно передать несколько событий ИИ для сжатия в один конспект, поддерживается двустороннее переключение между режимом конспекта и исходной временно́й линией
- Текст по умолчанию в редакторе системных промптов динамически обновляется в зависимости от включённых функций (например, при включении сети отношений в разделе формата тегов `<horae>` автоматически появляется строка `rel:`)

## Что нового в v1.7.0

Обновление добавляет систему векторной памяти, чтобы пользователи авто-конспекта больше не теряли детали из-за сжатия.

### Новые функции

- **Векторная память (Бета)**: умная система воспоминаний на основе локальных ИИ-моделей, специально разработанная для использования с «Авто-конспектом и скрытием». Когда диалог касается исторических событий, плагин автоматически извлекает релевантные фрагменты из скрытых старых сообщений и инъектирует в контекст
  - **Полностью локальные вычисления**: работает в браузере через Web Worker с лёгкой ИИ-моделью без дополнительного расхода API. При первом использовании необходимо скачать ~30–60 МБ модели (затем кэшируется в браузере)
  - **Два варианта модели**: `bge-small-zh-v1.5` (оптимизация для китайского) и `multilingual-e5-small` (многоязычная поддержка)
  - **Умный поиск**: три уровня — распознавание намерений, структурированный запрос и семантический поиск. Например, «упомянуть первую встречу с персонажем» напрямую найдёт первое появление этого персонажа без нечёткого сопоставления
  - **Контекстная связь**: соседние события вокруг найденного сообщения тоже включаются, восстанавливая полный контекст
  - **Полный обзор**: при высокой уверенности результата поиска может отправляться исходный текст (с автоматическим фильтром цепочки мышления), давая ИИ полную нарративную детализацию вместо краткого таймлайна. Число записей и порог настраиваются, установка 0 отключает функцию
  - **Изоляция конспектов**: данные конспектов автоматически исключаются — поиск ведётся только по исходным данным таймлайна, избегая повторной инъекции конспектов

## Что нового в v1.8.0

- **Режим RPG**: при включении ИИ выводит атрибуты персонажей и изменения навыков через теги `<horaerpg>`, не мешая существующим тегам `<horae>`
  - **Пользовательские полосы атрибутов**: по умолчанию HP/MP/SP, пользователь может свободно добавлять/удалять и менять имена и цвета (например, MP → «Духовная сила», добавление «Выносливость», «Желание» и т. д.)
  - **Отслеживание статусов**: поддержка десятков ключевых слов статусов — оглушение, кровотечение, заморозка, горение, окаменение и другие — с автоматическим подбором иконок
  - **Система навыков**: запись принадлежности навыка, уровня и описания эффекта; навыки связаны с NPC через номера, поддерживается ручное добавление и авто-генерация ИИ
  - **Нижняя HUD-панель**: над панелью сообщений отображаются упрощённые статус-полосы присутствующих персонажей с атрибутами и иконками статусов, автоматически подстраивается под ширину и смещение нижней панели
  - **Вкладка RPG**: на верхней панели появляется новая вкладка RPG с конфигурацией полос атрибутов, детальными атрибутами и статусами по каждому персонажу, списком навыков
  - **Условная инъекция**: при отключённом режиме RPG промпты не инъектируются совсем — нулевой расход токенов; отправка полос атрибутов и списка навыков независимо переключается
